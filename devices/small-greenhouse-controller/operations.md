# Operations

## Setup

- 鉢番号を `U1 U2 U3 / L1 L2 L3` で固定する。
- 温湿度センサーと CO2 センサーは上部中央のセンサーバーにまとめる。
- 土壌水分センサーは各鉢の代表点に挿し、棚ごとに束ねる。
- ポンプ6台、出力ハブ、12V電源は温室外のサービスボックスにまとめる。
- 制御ボックスは温室外の上側に固定し、ADCハブも同じ箱に入れる。
- `5V logic` と `12V actuator` の電源ラベルを明示する。
- 6系統すべてのチューブ先端を鉢番号に合わせてラベル付けする。

## Maintenance

- 週次でチューブ折れ、チューブ抜け、結露跡、配線擦れを確認する。
- 週次でサービスボックス内の端子緩みとポンプ漏れを確認する。
- 月次で各鉢の吐出量を再計測し、ポンプごとの差を見直す。
- 月次でセンサー値のドリフトを確認する。
- 月次で Arc 再接続と Harvest へのデータ到達を確認する。

## Failure handling

- 水分センサー異常時は、その鉢だけ自動給水を止める。
- ポンプ異常時は、その鉢だけを隔離する。
- 出力ハブ異常時は給水系だけを止め、環境計測と共有は継続する。
- 通信断時もローカル制御は継続する。

## Operational notes

- 同時給水は行わない。
- まず `上段3鉢` と `下段3鉢` の配線束が前面作業の邪魔にならないことを優先する。
- 交換頻度が高いのはセンサーよりポンプ側なので、ポンプ側のアクセスを優先する。
