# Control Notes

## Control loops

- 温度制御:
  - 上段/下段ファンを個別に扱えるようにする。
  - 高温側は送風、低温側は加温を使い分ける。
- CO2/換気制御:
  - CO2 低下時はまず循環で改善を見る。
  - 高 CO2 時は換気優先に倒す。
- 給水制御:
  - 6鉢を個別に判定する。
  - 同時給水しない。
  - 給水後は鉢ごとにクールダウン時間を持たせる。

## Recommended identifiers

- `U1 U2 U3`
- `L1 L2 L3`

## Data to keep

- 取得時刻
- 温室内温度
- 温室内湿度
- 温室内 CO2
- 各鉢の水分値
- 各ポンプの駆動イベント
- 上段/下段ファンの稼働状態
- アラームイベント

## Logic notes

- ポンプは `1回に1台` だけ動かす。
- 給水順は `乾きが強い鉢優先` にするが、同一鉢の連続駆動を避ける。
- 水分センサー6本は `ADCハブ` でまとめて読む。
- 出力は `8ch出力ハブ` にまとめ、Pico 側の GPIO 消費を減らす。
- 電源断復帰時は全出力 OFF から始める。
- Arc や Wi-Fi が落ちても、最後に確定している制御ルールだけでローカル運転を継続する。
